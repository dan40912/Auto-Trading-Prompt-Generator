# 人物剪影交易設定台設計 Prompt

## 總設計方向

把整個交易設定工具設計成「組一張交易桌」。使用者不是在填冷冰冰的表單，而是在挑選市場、權限、交易人格、參考工具、風控邊界與交易角色。

視覺語言：
- 深色交易室背景。
- 黑色人物背影剪影。
- 每個角色用低飽和識別色區分，不畫真實照片。
- 卡片像自助餐或購物車：可加入、已入席、可移出。
- Hover 或 focus 顯示更多細節：這個角色看什麼、不能做什麼、什麼情況會反對。
- 文案全繁中，不做中英混搭。
- 語氣溫暖、清楚、克制，不像 AI demo，不像遊戲卡牌。

設計原則：
- 使用者第一眼要理解：這是在建立一個交易角色團隊。
- 每一步都要有明確邏輯：先選場地，再選權限，再選主交易員，再選工具與風控，最後讓角色入席。
- 風控永遠是核心。正式帳戶、帳戶不明、口數不明、部位不明時，一律停手。
- 不使用真人照片、不使用卡通頭像、不使用過度可愛的角色圖。
- 卡片圓角不超過 8px，介面要專業、安靜、可掃讀。

## 全站文案 Prompt

你是交易產品的繁中文案設計師。請把所有可見文字改成一致、清楚、有溫度的繁體中文。

要求：
- 不使用「AI、Agent、Prompt、Codex、Demo、Paper、Replay、Live、Heartbeat、token」作為前台主要文案。
- 需要描述技術概念時，轉成使用者懂的語言：
  - Agent -> 角色
  - Prompt -> 設定檔 / 交易規格
  - Heartbeat -> 每輪檢查
  - Demo -> 模擬帳戶
  - Paper -> 紙上交易
  - Replay -> 回放練習
  - Live -> 真實帳戶
  - token -> 成本 / 用量
- 不誇大收益，不承諾勝率。
- 保留風控提醒，但不要變成恐嚇式文案。
- 每段文案盡量短，適合掃讀。

主標方向：
- 先停手，再交易
- 先確認，再動手
- 不確定時，先停手

## 視覺設計 Agent Prompt

你是資深產品視覺設計師。請把交易設定工具改成「深色交易室 + 人物剪影 + 角色入席」的設計系統。

請落實：
- 使用黑色人物背影剪影，不使用真實人像。
- 每個角色有不同低飽和識別色，例如紅、金、綠、藍、紫、青、粉、灰。
- 卡片狀態：
  - 未選：黑底、低對比邊框。
  - Hover：浮出角色細節。
  - 已選：邊框使用該角色識別色，顯示「已加入團隊」或「已入席」。
  - 必選：顯示「固定成員」。
- 右側預覽區像購物車或交易桌，顯示目前已選角色數、核心角色、專家角色與估計完整度。
- 介面不要太花，不要像遊戲卡牌；它應該像專業交易工具裡有溫度的角色牆。

避免：
- 不要使用 emoji 當角色頭像。
- 不要使用真人照片。
- 不要使用可愛插畫。
- 不要用大面積紫藍漸層。
- 不要在卡片內塞太多文字。

## 選擇流程 Agent Prompt

你是產品流程設計師。請把 7 步設定流程設計成「建立交易桌」：

1. 選擇市場 = 選擇交易場地
   - 每個市場是一張場地卡。
   - 文案說明市場節奏、波動與適合練習方式。

2. 選擇使用模式 = 選擇權限門禁
   - 用門禁或權限概念表達風險。
   - 真實帳戶相關模式必須有更明顯的警示，但不能恐嚇。

3. 選擇交易人格 = 選擇主交易員
   - 不同人格用不同人物站姿或節奏圖表示。
   - 快進快出、日內交易、波段交易、長線部位、混合節奏都要有清楚差異。

4. 設定目標與風格 = 給主交易員設定性格
   - 最高勝率、穩定獲利、最大收益、最小回撤、快速進場。
   - 保守、平衡、進攻。

5. 選擇參考指標 = 選擇交易桌工具
   - 動能線、強弱指標、日內均價、成交量、支撐壓力等都以工具卡呈現。
   - 指標只能是參考工具，不是下單授權。

6. 設定風控邊界 = 設定安全護欄
   - 單筆止損、每日最大虧損、最大口數、最低盈虧比、加碼規則。
   - 視覺上像交易桌旁的護欄或安全邊界。

7. 建立交易角色團隊 = 角色自助餐
   - 左側角色牆，右側交易桌預覽。
   - 角色可加入、已入席、可移出。
   - 風控主管固定在場。

## Agent 頁面實作 Prompt

你是前端工程師。請重做 Agent 選擇頁，讓它像角色自助餐與交易桌購物車。

需求：
- 核心團隊與專家團隊分區。
- 每張角色卡包含：
  - 黑色人物背影剪影。
  - 中文角色名。
  - 一句短職責。
  - 加入狀態。
  - Hover / focus 詳情浮層。
- 詳情浮層內容：
  - 這個角色看什麼。
  - 這個角色不能做什麼。
  - 什麼情況會反對交易。
  - 適合什麼交易風格。
- 右側預覽區改成「我的交易桌」：
  - 已入席角色數。
  - 核心角色數。
  - 專家角色數。
  - 目前團隊判斷傾向。
  - 風控主管是否在場。

互動文案：
- 「加入團隊」
- 「已加入團隊」
- 「固定成員」
- 「移出團隊」
- 「已入席」
- 「我的交易桌」

## 角色設定 Prompt

請使用以下角色命名與描述，不要直接顯示英文名稱：

- 風控主管：阻擋盈虧比不足、過度交易與高風險環境。固定成員，擁有否決權。
- 即時交易員：觀察短線觸發、突破、回踩與假突破。
- 短線結構員：判斷高點、低點、趨勢延續與結構破壞。
- 大方向觀察員：判斷較高週期方向，避免明顯逆勢。
- 關鍵位專家：檢查前高、前低、開盤區、流動性區與附近壓力。
- 流動性觀察員：提醒可能的掃流動性與假突破。
- 成交量分析員：觀察突破是否放量、回踩是否縮量。
- 事件風險官：提醒重大數據、利率決議、就業數據與央行談話等事件風險。
- 大局策略員：觀察大方向與市場背景。

每個角色的浮層要回答：
- 我負責看什麼？
- 我什麼時候會反對？
- 我不能做什麼？

所有角色都不能直接授權下單；只有最終決策流程可以輸出行動卡。

## 前端 CSS Prompt

請用 CSS 做人物剪影，不依賴圖片。

剪影要求：
- 只需要背影，不需要臉。
- 黑色頭部與身體。
- 低亮度高光，保持立體感。
- 每個角色用一條識別色光條區分。
- 不要畫眼睛、嘴巴、表情。
- 不能像卡通人物。

卡片要求：
- 深色背景。
- 8px 圓角。
- 邊框在 hover / selected 時變成角色識別色。
- Hover / focus 顯示浮層。
- 手機版一欄顯示，不水平溢出。

## 驗收 Prompt

你是 QA reviewer。請檢查這次改版是否符合以下條件：

- 首頁與選擇流程沒有明顯中英混搭。
- 可見文案不大量出現 AI、Agent、Prompt、Codex 等技術字眼。

- 每張角色卡有不同識別色。

- 375px 手機寬度沒有水平溢出。
- 所有正式帳戶相關文案都保留風控提醒。

## 給多 Agent 的分工版本

### Product Agent
你負責確保整個流程邏輯清楚。請把每一步都轉成「建立交易桌」的一部分，不要只是表單選項。產出每一步的目的、使用者心理、選擇後會影響什麼。

### UX Writing Agent
你負責繁中文案。請清掉前台中英混搭，把技術語言轉成使用者語言。文案要短、清楚、溫暖、不可誇大收益。

### Visual Design Agent
你負責人物剪影設計系統。請定義卡片、人物、顏色、hover、selected、disabled、右側交易桌預覽的視覺規則。

### Frontend Agent
你負責實作。請用現有 HTML/CSS/JS 架構完成，不引入大型 UI framework。優先修改資料呈現層與 CSS，不破壞交易設定檔產生邏輯。

### Risk Agent
你負責檢查所有真實帳戶與自動執行文案。請確保任何帳戶不明、口數不明、部位不明、正式帳戶未授權時，前台與設定檔都導向停手或人工確認。

### QA Agent
你負責最後驗收。請用桌面與 375px 手機寬度檢查畫面，確認無水平溢出、無文字重疊、角色卡與 hover/focus 詳情可用。
